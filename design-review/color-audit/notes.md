## 主要结论

整体颜色方向可以保留：浅灰页面背景、白色内容卡片、深海军蓝导航、金色小标签、绿色成功状态，适合“公开预测 + 成绩验证”的工具型站点。

需要优化的不是大面积换色，而是收敛强调色语义：

- 金色小标签 `#b28a18` 在白底上的对比度约 3.21:1，用在 10px 小字时偏弱。建议换深一点，例如复用旧 token `#735c00`，或把金色只用于背景/边框，小字改深色。
- 绿色同时用于赛场分组、结果确定位、收益/命中率等正向状态，语义容易混在一起。建议绿色只保留给成功/结果/收益；赛场分类改成蓝、靛、琥珀等非成功色，或统一为中性标签加细色条。
- 顶部导航当前项的高饱和蓝和阴影较强，尤其移动端首屏会抢主标题注意力。建议保留蓝色但减弱阴影，或改为深海军蓝底 + 细蓝下划线。
- 赛场颜色在 `RaceGrid.jsx` 和 `RaceDetailPage.jsx` 中重复硬编码，后续容易漂移。建议抽成同一份颜色配置或 CSS 变量。
- 管理登录页深色系可以保留为内部后台，但如果普通用户会看到入口，建议和公开站点共用同一组深海军蓝/金色/绿色变量，减少品牌割裂。

## 最高优先级问题

关于页首屏不是配色本身的问题，而是布局把左侧文案压成竖排，导致巨大白色空洞。这个问题会让任何颜色优化都显得无效。

建议先修：

- `public-static-page__hero` 不要使用 `minmax(0, 1fr) auto` 搭配三列 meta。
- 右侧 meta 区建议限制宽度，例如 `minmax(360px, 520px)`。
- 或者把 meta 放到 hero 下方，桌面也使用一行三卡，避免压缩标题和 lead。

## 建议优先级

1. 先修关于页 hero 布局。
2. 加深金色文字 token，保证小标签可读。
3. 收敛绿色语义，避免赛场分类和成功状态冲突。
4. 降低顶部当前导航的蓝色阴影强度。
5. 合并赛场色板配置。
